# SOP-003: Change Management — Quản lý Thay đổi

> **Actor**: COO + CEO | **Trigger**: Thay đổi quy trình, cấu trúc, công cụ, hoặc chiến lược

## Khi nào áp dụng
- Thay đổi cấu trúc tổ chức (thêm/bớt phòng ban, thay đổi reporting line)
- Triển khai công cụ/hệ thống mới
- Thay đổi quy trình vận hành
- Chuyển đổi chiến lược (pivot)

## Quy trình ADKAR

### 1. AWARENESS — Nhận thức
```
// turbo
├── WHY: Tại sao cần thay đổi? (data-driven, không phải "vì muốn")
├── WHAT IF NOT: Hậu quả nếu KHÔNG thay đổi?
├── Communicate: CEO/COO thông báo cho toàn tổ chức
└── Địa chỉ lo ngại: Q&A session
```

### 2. DESIRE — Mong muốn
```
// turbo
├── WIIFM: What's In It For Me? → Lợi ích cho TỪNG phòng ban
├── Champions: Chọn 1-2 người ủng hộ sớm trong mỗi dept
├── Quick wins: Show kết quả tích cực sớm nhất có thể
└── Address resistance: Lắng nghe, không ép buộc
```

### 3. KNOWLEDGE — Kiến thức
```
// turbo
├── Training plan: Ai cần học gì?
├── Documentation: SOP mới, guides, FAQs
├── Hands-on practice: Sandbox/pilot environment
└── L&D Specialist hỗ trợ thiết kế training
```

### 4. ABILITY — Năng lực
```
// turbo
├── Pilot: Thử với 1 team/dept trước (2-4 tuần)
├── Support: Help desk cho giai đoạn chuyển đổi
├── Iterate: Thu feedback, điều chỉnh trước khi rollout toàn bộ
└── Measure: Adoption rate, productivity, satisfaction
```

### 5. REINFORCEMENT — Củng cố
```
// turbo
├── Celebrate: Công nhận teams adopt thành công
├── Monitor: Track metrics 30/60/90 ngày
├── Feedback loop: Regular check-ins
├── Adjust: Fine-tune dựa trên thực tế
└── Document: Cập nhật SOPs, COMPANY_CONTEXT.md
```

## Impact Assessment Matrix

| Change Type | Impact | Approval | Lead Time |
|-------------|--------|---------|-----------|
| Process (minor) | 1 dept | COO | 1 tuần |
| Process (major) | Multi-dept | CEO + COO | 2-4 tuần |
| Tool/System | Tech + Users | CTO + COO | 4-8 tuần |
| Org Structure | All | CEO + Board | 4-12 tuần |
| Strategy Pivot | All | CEO + Board | Variable |
