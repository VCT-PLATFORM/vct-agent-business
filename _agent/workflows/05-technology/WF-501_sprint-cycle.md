# WF-501: Sprint Cycle

> Workflow: **PM → Tech Lead → Dev → QA → Stakeholders**

```
Week 1                              Week 2
Plan → Daily Dev → PR Review → QA → Demo → Retro
PM+TL   Dev+TL      TL+Dev    QA   PM+Stkh  Team
```

## 2-Week Sprint Flow
```
// turbo
Day 1 (Monday W1): SPRINT PLANNING (2h)
├── PM presents prioritized backlog (RICE scored)
├── Tech Lead leads estimation (story points)
├── Team commits to sprint scope
├── Sprint Goal defined (1 sentence)
└── Board updated: To Do → In Progress →  → Done

Daily (15 min standup):
├── What I did yesterday
├── What I'll do today
├── Any blockers? → TL resolves within 4h

Day 5 (Friday W1): MID-SPRINT CHECK
├── Burndown on track?
├── Any scope risk? → Negotiate with PM
├── PR backlog?

Day 8-9 (W2): CODE COMPLETE + QA
├── All PRs submitted by Day 8 EOD
├── Code review: SOP-404 (Code Review)
├── QA testing: SOP-401 Phase 5
├── Bug fixes Day 9

Day 10 (Friday W2): SPRINT CLOSE
├── DEMO (30 min): Show working features to stakeholders
│   ├── PM presents context
│   ├── Dev demos features
│   └── Stakeholders ask questions, give feedback
├── RETRO (30 min): Start / Stop / Continue (SOP-504)
├── DEPLOY: If approved → SOP-403 (Release)
```
