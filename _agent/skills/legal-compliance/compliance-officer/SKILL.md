## Chuyên môn

### Data Privacy (NĐ 13/2023 — VN PDPA)
```
Key Requirements:
├── Consent: Phải có đồng ý trước khi thu thập dữ liệu cá nhân
├── Purpose Limitation: Chỉ dùng đúng mục đích đã thông báo
├── Data Minimization: Chỉ thu thập dữ liệu CẦN THIẾT
├── Storage Limitation: Không giữ lâu hơn cần thiết
├── Cross-border Transfer: Cần Impact Assessment nếu chuyển data ra nước ngoài
├── Data Subject Rights: Access, Rectification, Deletion, Withdrawal
├── Breach Notification: 72h thông báo Bộ Công an nếu có vi phạm
└── Data Protection Impact Assessment (DPIA): Khi xử lý dữ liệu nhạy cảm
```

### Compliance Program Framework
```
1. ASSESS: Identify applicable regulations & risks
2. DESIGN: Policies, procedures, controls
3. IMPLEMENT: Training, tools, processes
4. MONITOR: Audits, testing, metrics
5. REPORT: Board reporting, regulatory filings
6. IMPROVE: Lessons learned, continuous improvement
```

### Privacy by Design Checklist
- [ ] Data mapping: Biết data gì, ở đâu, ai access
- [ ] Consent mechanism: Clear, granular, easy to withdraw
- [ ] Privacy policy: Đầy đủ, dễ hiểu, accessible
- [ ] Data retention schedule: Bao lâu xóa data
- [ ] Encryption: At rest + In transit
- [ ] Access controls: Principle of least privilege
- [ ] Vendor assessment: Third parties handle data properly
- [ ] Incident response plan: Ready for data breach
- [ ] Training: Annual privacy awareness training
